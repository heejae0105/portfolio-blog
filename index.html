01. IoT 다인 탑승 감지 시스템
Arduino C++ Sensor Fusion

"킥보드 2인 탑승, 진짜 못 막을까?"라는 호기심에서 시작해, 압력 센서와 초음파 센서를 융합하여 오탐률을 획기적으로 줄인 하드웨어 프로젝트입니다.

개발 과정 및 상세 내용 보기 (Click)

1. 문제 정의 (Why?)

전동 킥보드 2인 탑승 사고가 급증하고 있지만, 기존의 단순 무게 감지 방식은 무거운 짐을 실었을 때도 경고가 울리는 오작동 문제가 있었습니다. 이를 해결하기 위해 물리적 조건(무게)과 공간적 조건(부피)을 동시에 판단하는 시스템이 필요하다고 생각했습니다.

2. 해결 솔루션 (How?)

Sensor Fusion (센서 융합): 단일 센서의 한계를 극복하기 위해 두 가지 데이터를 교차 검증하는 알고리즘을 설계했습니다.

  • 조건 1 (FSR 압력 센서): 발판의 4지점 중 3곳 이상에서 압력이 감지되는가? (사람의 발 위치 분석)
  • 조건 2 (초음파 센서): 탑승자의 전/후방 거리가 기준치 이하로 줄어들었는가? (밀집도 분석)
  • 판단 로직: 위 두 조건이 동시에 3회 이상(Cycle) 감지될 때만 '위험'으로 판단하여 경고음을 송출합니다.

3. 배운 점 & 성과

센서 데이터에는 항상 노이즈(튀는 값)가 존재한다는 것을 깨달았습니다. 이를 소프트웨어적으로 필터링하는 과정을 통해, 현실 세계의 데이터를 다루는 법을 배웠습니다.

02. P2P 블록체인 메신저
Python Socket Blockchain

카카오톡 서버 다운 사태를 보며 "중앙 서버가 없어도 대화할 수 없을까?"를 고민했습니다. Python 소켓 통신과 블록체인 원리를 직접 구현해본 도전적인 프로젝트입니다.

개발 과정 및 상세 내용 보기 (Click)

1. 탐구 배경

대부분의 메신저는 중앙 서버(SPOF)에 의존합니다. 서버가 터지면 모든 소통이 마비되는 구조적 취약점을 해결하기 위해, Peer-to-Peer(P2P) 방식의 탈중앙화 메신저를 구상했습니다.

2. 핵심 기술 구현

  • Socket Programming: 서버를 거치지 않고 Client A와 Client B가 직접 패킷을 주고받는 통신 환경을 구축했습니다.
  • Data Integrity (무결성): 이전 메시지의 해시값을 현재 메시지에 포함시키는 블록체인 구조(Chain)를 적용했습니다. 누군가 대화 내용을 몰래 수정하면 체인이 끊겨 즉시 발각되도록 설계했습니다.

3. 결론

단순히 남들이 만든 API를 쓰는 게 아니라, 네트워크 통신의 가장 밑바닥인 소켓 레벨부터 직접 짜보면서 데이터가 어떻게 흘러가는지 확실하게 이해하게 되었습니다.

03. DealChecker: AI 계약서 분석
AI / NLP Fine-Tuning Web

어려운 법률 용어 때문에 피해를 보는 사람이 없도록, AI를 학습시켜 계약서의 독소 조항을 찾아내는 웹 서비스입니다. 기술의 사회적 가치를 실현해보고 싶었습니다.

개발 과정 및 상세 내용 보기 (Click)

1. 프로젝트 개요

법률 전문가가 아니면 계약서의 '불공정 조항'을 찾아내기 어렵습니다. 이를 해결하기 위해 자연어 처리(NLP) 기술을 활용해 계약서를 자동 분석하고 위험도를 시각화하는 플랫폼을 개발했습니다.

2. 기술적 챌린지

일반적인 언어 모델은 법률 전문 용어에 약했습니다. 이를 해결하기 위해 불공정 약관 데이터셋을 확보하여 모델을 파인튜닝(Fine-Tuning)했고, 분석 정확도를 높이는 과정에서 AI 학습 파이프라인 전반을 경험했습니다.

3. 의의
